# Contributing to <PROJECT NAME>

Thanks for taking the time to contribute! Please follow these guidelines to help us keep things consistent and easy to review.

## Code of Conduct
By participating, you agree to abide by our [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md).

## Getting Started (GitHub UI)
- **Fork** the repository to your account from the repository page on GitHub.
- Create a **new branch** from the Branch dropdown on GitHub (e.g., `feature/short-description`).
- If you work on the web editor or GitHub Codespaces/GitHub Desktop, make sure your changes are saved and committed with a clear message.

## Making Changes
- Keep changes focused and small.
- Update documentation (README, comments) when behavior changes.
- Add/adjust tests where applicable (if the project uses tests).

## Commit Messages
- Use clear, present-tense messages with a short summary line (about 72 chars).
- Examples:
  - `fix: correct typo in README`
  - `feat: add validation to simple-interest.sh`
  - `docs: clarify setup steps`

## Pull Requests (GitHub UI)
- Open a **Pull Request** from your branch into `main` using the "Compare & pull request" button on GitHub.
- Ensure:
  - [ ] The project builds/passes checks (if any)
  - [ ] Style/lint checks pass (if any)
  - [ ] Docs updated when needed
- In the PR description, explain the motivation, what changed, and how to verify.
- Link related issues (e.g., `Closes #123`).

## Issue Reporting
For bugs, include:
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, versions) and logs if relevant

For features, include:
- Motivation / use case
- Proposed solution / alternatives
